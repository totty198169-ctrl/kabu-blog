# CLAUDE.md — 執筆ガイド

このリポジトリは Astro 製の株式投資ブログです。記事を作成・編集する際は、本ファイルの規約に従ってください。

## 記事の保存場所と命名規則

- 保存先: `src/content/blog/`(`.md` または `.mdx`)
- ファイル名: `YYYY-MM-DD-<英小文字スラッグ>.md`
  - 例: `src/content/blog/2026-06-06-fujitsu-bunseki.md`
  - スラッグは半角英小文字・数字・ハイフンのみ。日本語やスペースは使わない。
  - ファイル名がそのまま URL になる(`/blog/2026-06-06-fujitsu-bunseki/`)。
- 下書きは frontmatter に `draft: true` を付ける(一覧・個別ページ・RSS から除外される)。

## フロントマター(必須項目)

| キー | 必須 | 型 | 説明 |
| --- | --- | --- | --- |
| `title` | ✅ | string | 記事タイトル |
| `pubDate` | ✅ | date | 公開日(`YYYY-MM-DD`)。日付フィールド名は `date` ではなく **`pubDate`** |
| `category` | ✅ | enum | 下記4カテゴリのいずれかを**正確に**記述 |
| `tags` | ✅ | string[] | 1つ以上付ける |
| `description` | ✅ | string | 一覧・OGP・RSS に使う要約(1〜2文) |
| `ticker` | 任意 | string | 主対象の証券コード(例: `"6702"`) |
| `updatedDate` | 任意 | date | 更新日 |
| `draft` | 任意 | boolean | 下書きフラグ(既定 `false`) |

> スキーマ上 `description` / `tags` は省略可ですが、本ブログでは**必須運用**とします。カテゴリ名を誤るとビルドエラーで検知されます。

### カテゴリ(4種・固定)

`スイングトレード` / `高配当` / `デイトレ` / `銘柄分析`

(定義は `src/content.config.ts` の `CATEGORIES`、URL スラッグは `src/consts.ts` の `CATEGORY_SLUGS`)

### frontmatter 例

```markdown
---
title: 富士通(6702)の中期業績と株価水準を分析
pubDate: 2026-06-06
category: 銘柄分析
tags: [決算, 半導体, 中期経営計画]
description: 富士通(6702)の直近業績とバリュエーションを整理し、想定シナリオを記録する。
ticker: "6702"
---
```

## 記事の標準構成

本文は以下の順序で書く(見出しは `##` を使用):

1. **導入** — 何について書くか、背景・きっかけを2〜3文で。
2. **結論先出し** — 見立て・スタンスを最初に明示する(ただし断定推奨は避ける、下記参照)。
3. **根拠** — テクニカルとファンダの両面から。
   - テクニカル: 移動平均・出来高・チャート形状・需給 等
   - ファンダ: 業績・指標(PER/PBR/利回り等)・カタリスト 等
4. **リスク** — 想定が崩れる条件、下振れ要因、損切り・撤退ラインを必ず明記。
5. **まとめ** — 要点の再整理と今後の確認ポイント。

## 銘柄・表現に関するルール

- **証券コードを必ず併記する。** 銘柄名に初出で `企業名(コード)` の形を付ける。例: `トヨタ自動車(7203)`。`ticker` にも記載する。
- **断定的な推奨表現は使わない。** 「買い」「必ず上がる」「絶対」「狙い目」等の断定・煽り表現は避ける。
  - 推奨されない例: 「6702 は今が買い」「確実に上昇する」
  - 推奨される表現: 「〜と考える」「〜の可能性がある」「あくまで筆者の見立て」「想定シナリオの一つ」
- 一次情報(適時開示 TDnet・有価証券報告書・決算短信等)を根拠とし、可能な範囲で出所を示す。
- 数値・指標は記事内に**日付**を添える(時点が変わると意味が変わるため)。

## 免責文の扱い

- 免責文は全ページ共通フッター(`src/components/Footer.astro`)に常設しており、**記事本文に毎回書く必要はない**。
- フッターの定型文(「投資は自己責任で」を含む)は本ブログの基本方針。安易に削除・改変しない。
- 個別記事でリスクが特に大きいテーマ(信用取引・イベント投資・新興株など)は、本文の「リスク」節で具体的な注意喚起を追記する。
- 記事はすべて筆者個人の記録・見解であり、特定銘柄の売買を推奨するものではない、という前提を崩さない。
